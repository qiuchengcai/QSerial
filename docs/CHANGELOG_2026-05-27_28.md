## 一、Phase 1 — MCP SDK 迁移与安全加固

> **提交:** `670ea66` | 2026-05-27 22:15 | 12 files, +2120/-364

### 1.1 MCP SDK 替换

**变更前**：项目使用手写的 MCP 协议实现，自行处理 JSON-RPC 解析、SSE 推送、速率限制等逻辑。

**变更后**：引入官方 `@modelcontextprotocol/sdk` v1.29.0，替换手写实现。

移除的死代码：
- `handleRpc()` — 自定义 RPC 分发器
- `notifySse()` — 自定义 SSE 广播
- `sendSse()` — SSE 发送逻辑
- `checkRateLimit()` — 速率限制检查

这些功能全部由 SDK 内置的 `StreamableHTTPServerTransport` 和 SSE 机制接管，减少维护负担。

### 1.2 安全加固

### 6.1 MCP Sampling（采样）

实现 MCP 协议中的 **Server-initiated Sampling** 能力——服务器可以主动向 AI 客户端发起决策请求。

新增 `services/mcp/sampling.ts`：

| 触发场景 | 决策选项 | 说明 |
|----------|----------|------|
| `connection_login` 未知提示符 | `retry` / `send_anyway` / `abort` | 登录遇到未识别的提示符时请求 AI 判断 |
| `connection_run_script` 步骤失败 | `retry` / `skip` / `abort` | 脚本执行失败时请求 AI 决策 |

工作流程：
```
设备输出 ──→ 不匹配已知模式 ──→ 触发 Sampling 请求
                                      │
                    ┌─────────────────┼─────────────────┐
                    ▼                 ▼                  ▼
                retry (重试)    send_anyway/skip    abort (终止)
```

Capabilities 更新为 `{tools, resources, sampling}`。
