# QSerial 测试指南

## 测试环境准备

### 串口测试

#### Windows
使用虚拟串口软件创建测试环境：
- **com0com** - 开源虚拟串口驱动
- **Virtual Serial Port Driver** - 商业软件

安装 com0com：
1. 下载并安装 com0com
2. 创建一对虚拟串口（如 COM3-COM4）
3. 一个用于 QSerial 连接，另一个用于测试数据发送

#### Linux/macOS
可以使用 socat 创建虚拟串口：
```bash
socat -d -d pty,link=/tmp/ttyV0,raw pty,link=/tmp/ttyV1,raw
```

### SSH 测试

#### 本地 SSH 服务器
使用 Docker 快速搭建：
```bash
docker run -d -p 2222:22 --name ssh-test \
  -e PUID=1000 -e PGID=1000 \
  -e TZ=Asia/Shanghai \
  linuxserver/openssh-server
```

连接测试：
```bash
ssh -p 2222 testuser@localhost
```

#### 远程服务器
使用已有的 SSH 服务器进行测试。

## 功能测试

### 1. 串口连接测试

#### 测试步骤
1. 打开 QSerial 面板（侧边栏）
2. 点击"连接"选项卡
3. 点击串口设备刷新按钮
4. 选择一个串口，右键点击"连接"
5. 选择波特率（默认 115200）
6. 观察终端是否打开

#### 验证要点
- 串口连接成功后状态栏显示连接信息
- 树状图中串口状态变为"已连接"
- VS Code 终端窗口打开

### 2. SSH 连接测试

#### 密码认证测试
1. 点击"新建 SSH 连接"
2. 输入主机地址、端口、用户名
3. 选择"密码认证"
4. 输入密码
5. 观察连接是否成功

#### 私钥认证测试
1. 确保私钥文件存在（~/.ssh/id_ed25519 或 id_rsa）
2. 点击"新建 SSH 连接"
3. 输入主机信息
4. 选择"私钥认证"
5. 如有 passphrase 则输入
6. 观察连接是否成功

#### 多连接测试
1. 连接多个 SSH 服务器
2. 验证每个连接独立工作
3. 测试断开单个连接不影响其他

### 3. 数据收发测试

#### 串口数据测试
1. 连接虚拟串口对的一端（如 COM3）
2. 使用另一端（COM4）发送数据
3. 观察 QSerial 终端是否正确显示
4. 在 QSerial 终端输入数据
5. 观察 COM4 是否收到数据

#### 编码测试
测试不同编码的数据显示：
- GBK 中文数据
- UTF-8 数据
- HEX 模式显示

### 4. 快捷按钮测试

#### 创建按钮
1. 切换到"快捷按钮"选项卡
2. 点击添加按钮
3. 输入按钮名称、命令
4. 选择目标（串口/SSH/两者）

#### 执行按钮
1. 连接终端
2. 点击快捷按钮
3. 观察命令是否正确发送

#### 多命令序列
1. 创建包含多个命令的按钮
2. 设置命令间延迟
3. 执行并观察命令序列执行

### 5. 日志记录测试

1. 连接终端后右键点击
2. 选择"开始记录日志"
3. 发送/接收数据
4. 选择"停止记录日志"
5. 打开日志文件夹查看日志文件

### 6. MCP 功能测试

参考 `docs/MCP_SETUP.md` 配置 MCP Server。

#### 测试 MCP 工具
```bash
# 编译项目
npm run compile

# 启动 MCP Server
npm run mcp
```

## 问题排查

### 串口无法连接
1. 检查串口是否存在
2. 检查是否有其他程序占用
3. 检查波特率设置

### SSH 连接失败
1. 检查网络连通性
2. 检查用户名/密码
3. 检查私钥文件权限
4. 查看 SSH 服务器日志

### 数据显示乱码
1. 检查编码设置
2. 尝试不同编码选项
3. 检查数据源编码

### MCP 无法启动
1. 确保已编译 `npm run compile`
2. 检查 MCP 配置路径
3. 查看 MCP Server 日志
