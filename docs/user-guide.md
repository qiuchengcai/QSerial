## 概述

QSerial 是一款现代化的跨平台终端工具，主要面向嵌入式开发者、运维工程师和开发者，集串口调试、远程连接、终端模拟、文件传输于一体。

**核心功能**：

- 多协议连接（串口、SSH、Telnet、本地终端）
- 连接共享（TCP 局域网共享 + SSH 反向隧道 + TELNET 协议协商）
- SFTP 文件传输（基于 SSH 的文件浏览器）
- TFTP 服务器（嵌入式设备固件传输）
- FTP 服务器（标准文件共享）
- NFS 服务器（Windows/Linux 网络文件系统）
- 多标签管理（拖拽排序、分组管理）
- 主题定制（8 套预设）
- 快捷按钮（多行命令 + 行间延迟）

## 安装与启动

### 安装

从 [Release](https://github.com/qserial/qserial) 下载对应平台安装包：

| 平台 | 格式 |
|------|------|
| Windows | 安装包 + 便携版 |
| Linux | AppImage + deb |
| macOS | DMG |

### 从源码构建

```bash
# 环境要求: Node.js >= 18, pnpm >= 8.15

pnpm install                # 安装依赖
pnpm build:shared           # 构建共享包
pnpm dev                    # 启动开发模式
pnpm build                  # 构建所有包
pnpm package:win            # 打包 Windows
pnpm package:linux          # 打包 Linux
pnpm package:mac            # 打包 macOS
```

## 连接类型

QSerial 支持 6 种连接类型，均在侧边栏点击对应按钮创建：

| 连接类型 | 说明 | 典型场景 |
|----------|------|----------|
| **本地终端 (PTY)** | 本机 Shell | 日常命令行操作 |
| **串口 (Serial)** | 串口连接 | 嵌入式设备调试 |
| **SSH** | SSH 远程连接 | 服务器运维 |
| **Telnet** | Telnet 远程连接 | 网络设备管理 |
| **连接共享 (ConnectionServer)** | TCP 共享任意活跃连接（推荐） | 远程访问任意终端会话 |
| **串口共享 (SerialServer)** | TCP 共享串口（已废弃，使用连接共享代替） | — |

### 串口连接

1. 点击侧边栏「串口」按钮
2. 选择串口路径（自动检测可用端口）
3. 配置参数：

| 参数 | 默认值 | 可选值 |
|------|--------|--------|
| 波特率 | 9600 | 300 ~ 921600 |
| 数据位 | 8 | 5, 6, 7, 8 |
| 停止位 | 1 | 1, 1.5, 2 |
| 校验位 | none | none, even, odd, mark, space |

4. 点击「连接」

串口断开后可自动重连（默认开启，间隔 3 秒，最多 5 次）。

### SSH 连接

1. 点击侧边栏「SSH」按钮
2. 填写连接信息：

| 参数 | 说明 |
|------|------|
| 主机 | 目标服务器地址 |
| 端口 | 默认 22 |
| 用户名 | 登录用户名 |
| 认证方式 | 密码 或 密钥 |

3. 点击「连接」

SSH 连接内置 keepalive 保活（默认 30 秒间隔），兼容老旧设备的加密算法。

### Telnet 连接

1. 点击侧边栏「Telnet」按钮
2. 填写主机和端口
3. 点击「连接」

### 本地终端

点击侧边栏「本地终端」按钮即可打开本机 Shell。

## 连接共享

QSerial 的核心差异化功能：将任意活跃连接通过 TCP 网络共享给远程设备。支持两种模式：**连接共享**（推荐，共享任意类型连接）和串口共享（已废弃）。

### 工作原理

```
远程设备 ──telnet──→ [TCP/SSH] ──→ 本地 TCP Server ──→ 写入队列 ──→ 连接
                                               ↓
连接.onData ──→ 广播给已认证客户端 ──→ 远程设备
```

### 配置步骤

1. 点击侧边栏「连接共享」按钮
2. 配置共享参数：

| 参数 | 说明 | 默认值 |
|------|------|--------|
| 连接来源 | 选择已有连接或创建新连接作为数据源 | — |
| 本地端口 | TCP 监听端口 | 8888 |
| 监听地址 | `0.0.0.0` 局域网可访问，`127.0.0.1` 仅本机 | `0.0.0.0` |
| 访问密码 | 为空则无需认证 | — |
| SSH 隧道 | 可选，将连接共享到远程服务器 | — |

3. SSH 隧道配置（可选）：

| 参数 | 说明 |
|------|------|
| SSH 主机 | 远程服务器地址 |
| SSH 端口 | 默认 22 |
| 用户名 | SSH 登录用户名 |
| 远程端口 | 远程服务器上的监听端口 |
| 密码/密钥 | SSH 认证方式 |

SSH 隧道会自动尝试 `~/.ssh` 下的默认密钥（id_ed25519, id_rsa, id_ecdsa, id_dsa），断线自动重连（指数退避，最多 5 次）。

4. 点击「开始共享」

### 远程客户端连接方式

```bash
# 局域网连接（无密码）
telnet <QSerial所在机器IP> 8888

# 局域网连接（有密码）
telnet <QSerial所在机器IP> 8888
# 连接后直接输入密码回车即可认证

# SSH 隧道模式（在远程服务器上）
telnet localhost <远程端口>

# Windows（需先启用 Telnet 客户端）
dism /online /Enable-Feature /FeatureName:TelnetClient
telnet <QSerial所在机器IP> 8888

# 也可使用 socat 交互操作
socat - localhost:8888
```

> 推荐使用 telnet 客户端连接，支持完整的终端交互功能（Tab 补全、方向键等）。

### 密码认证协议

1. 客户端连接后，服务端发送 `PASSWORD: ` 提示
2. 客户端需在 **10 秒内**发送 `PASSWORD:xxx\n`（或直接输入密码）
3. 认证成功返回 `OK`，失败返回 `AUTH_FAILED`
4. 3 次失败后自动断开连接
5. 未认证客户端不会收到任何数据

## SFTP 文件传输

基于 SSH 连接的文件浏览器，连接 SSH 后自动可用。

### 功能

| 操作 | 说明 |
|------|------|
| 浏览目录 | 目录优先、名称排序 |
| 上传文件 | 本地 → 远程，带进度 |
| 下载文件 | 远程 → 本地，带进度 |
| 创建目录 | 在当前路径下新建 |
| 删除 | 删除文件或目录 |
| 重命名 | 文件/目录重命名 |
| 符号链接 | 创建/读取符号链接 |

### 使用方式

1. 先建立 SSH 连接
2. 点击终端面板上方的 SFTP 图标打开文件浏览器
3. 左侧本地文件，右侧远程文件，拖拽或右键操作

## TFTP 服务器

内置 TFTP 服务器，支持嵌入式设备的固件/文件传输。

### 配置步骤

1. 菜单 → TFTP 服务器（或对应按钮）
2. 设置参数：

| 参数 | 默认值 | 说明 |
|------|--------|------|
| 端口 | 69 | TFTP 监听端口 |
| 共享目录 | — | 文件存放的根目录 |

3. 点击「启动」

### 功能

- 文件上传（PUT）和下载（GET）
- 传输进度监控（每 5% 更新一次）
- 传输参数优化：blockSize=65464、windowSize=64，大幅提升传输速率
- 传输事件：started / progress / completed / aborted / error

---

## FTP 服务器

内置 FTP 服务器，基于 ftp-srv 实现，支持标准的文件上传和下载。

### 配置步骤

1. 菜单 → FTP 服务器（或侧边栏对应按钮）
2. 设置参数：

| 参数 | 说明 |
|------|------|
| 端口 | FTP 监听端口（默认 21） |
| 共享目录 | 提供文件访问的根目录 |
| 用户名 | FTP 登录用户名（可为空，匿名访问） |
| 密码 | FTP 登录密码 |

3. 点击「启动」

### 功能

- 文件上传和下载，带进度显示
- 匿名/用户认证模式
- 客户端连接状态实时监控
- 传输事件：started / progress / completed / error

---

## NFS 服务器

内置 NFS 服务器，支持 Windows（WinNFSd）和 Linux（nfs-kernel-server）双平台。

### 配置步骤

1. 菜单 → NFS 服务器（或侧边栏对应按钮）
2. 设置参数：

| 参数 | 说明 |
|------|------|
| 共享目录 | 要导出的本地目录 |
| 允许的客户端 | 允许挂载的客户端 IP 或网段 |
| 导出选项 | NFS 导出选项 |

3. 点击「启动」

### 功能

- 自动选择平台后端（Windows: WinNFSd / Linux: exportfs）
- 客户端连接/断开实时监控
- 挂载命令提示（自动生成本机 IP + 挂载命令，方便远程客户端挂载）

## 快捷按钮

终端下方可配置快捷按钮栏，用于快速发送常用命令。

### 功能

- **分组管理**：创建/编辑/删除/排序分组，下拉切换
- **自定义按钮**：名称、命令（支持多行逐条发送）、行间延迟、描述、颜色
- **预设颜色**：10 种预设 + 自定义颜色
- **右键菜单**：编辑/删除按钮或分组
- **持久化**：自动保存到本地

### 多行命令

按钮支持多行命令，点击后逐条发送：

- 在编辑弹窗中使用 textarea 输入多行命令，每行一条
- 可配置行间延迟（默认 100ms），适用于需要等待设备响应的场景
- 示例：登录设备 → 输入密码 → 执行命令，三行分别填写，行间延迟 500ms

## AI 设备操控（规划中）

> ⚠️ **此功能尚未实现，以下为设计规划。**

QSerial 的终端共享功能将为 AI 操控设备提供天然通道，计划通过 Skill 方案直连设备，无需 MCP 中间层。

### 规划架构

```
AI → execute_command → Python脚本 → TELNET → QSerial共享 → 串口 → 设备
```

### Skill vs MCP（设计对比）

| 维度 | Skill 方案 | MCP |
|------|-----------|-----|
| 执行链路 | 最短，无中间层 | 多一层进程间通信 |
| 响应速度 | 快，2-3 秒/命令 | 慢，JSON-RPC 封装/解析有额外耗时 |
| 资源开销 | 按需执行，零常驻 | Server 长驻进程 |
| Agent 搭建 | 即插即用，放入 skills 目录即可 | 需注册配置、启动参数 |
| Skill 分发 | 随仓库提交，clone 即可用 | 每台机器单独配置 |

## 会话管理

### 保存连接配置

1. 建立连接后，右键标签 → 「保存会话」
2. 侧边栏「已保存」列表中出现该配置

### 快速重连

点击侧边栏已保存的配置即可重新连接。

- 绿色圆点 = 已连接
- 空心圆 = 未连接
- 右键可编辑/删除配置

---

## 日志记录

1. 终端面板右键菜单 → 「开始日志」
2. 选择保存位置（.txt / .log）
3. 终端输出实时写入日志文件
4. 日志首尾带时间戳标记
5. 关闭日志或退出应用时自动停止
