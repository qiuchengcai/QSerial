## xterm.js `dimensions` 未定义错误分析
- **问题**：终端连接时每次报 `TypeError: Cannot read properties of undefined (reading 'dimensions')` at xterm.js:1776
- **根因**：xterm.js RenderService 的 `dimensions` getter 访问 `this._renderer.value.dimensions`，`_renderer` 是 `MutableDisposable`，dispose 后 `value` 返回 `undefined`
- **崩溃链路**：
  1. 写数据 → `xterm.write()` → 触发 scroll → `onScroll` → `syncScrollArea()` → `_refresh(false)` → `rAF(() => _innerRefresh())` 被调度
  2. React 组件 cleanup → `xterm.dispose()` → RenderService dispose → `_renderer._isDisposed = true`
  3. Viewport 未在 dispose 中 cancel `_refreshAnimationFrame`（Viewport 没有重写 `dispose()`）
  4. rAF 在已 dispose 的 Viewport 上触发 → `_renderer.value` 为 `undefined` → 崩溃
- **修复方向**：
  - `onData` 回调写入前增加 `disposedRef.current` 守卫
  - cleanup 中 dispose 前取消可能的外部 rAF 引用
  - 或在 `xterm.write()` 前后增加 disposed 检查
- **已修复**（TerminalPane.tsx，共4处改动，两次迭代）：
  1. `openXterm()` 中 patch `viewport.syncScrollArea` + `viewport._refresh`，用 `mountCountRef.current !== capturedMountId` 守卫（v1 误用 `disposedRef.current` → v2 修正为 `mountCountRef`——新 mount 会重置 disposedRef）
  2. `onData` 回调前加 `if (disposedRef.current) return;`
  3. `onStateChange` 回调前加 `if (disposedRef.current) return;`
  4. cleanup 中增加 `fitAddonRef.current = null;`

## Git Bash 终端创建失败修复
- 用户反馈创建 Git Bash 终端时报 `File not found:` 错误
- **根因**：Electron 主进程运行在 Windows 环境下，其 PATH 不包含 `C:\Program Files\Git\bin\`（Git Bash 的 bash.exe 所在目录）。`node-pty` 的 `spawn('bash')` 在 Windows PATH 中找不到该可执行文件
- **修复**：在 `pty.ts` 新增 `resolveShellPath()` 函数，在 `pty.spawn()` 之前解析 shell 的完整路径：
  - bash：优先查 `C:\Program Files\Git\bin\bash.exe` 等常见位置 + 注册表 `HKLM\SOFTWARE\GitForWindows` + `where` 命令
  - wsl：查 `C:\Windows\System32\wsl.exe`
  - powershell/cmd：查 `C:\Windows\System32\` 下对应路径
  - 如果已经是绝对路径，直接校验
- 文件：`packages/main/src/connection/pty.ts` + 手动同步 `packages/main/dist/connection/pty.js`
  
## xterm.js dimensions 错误修复（第三轮迭代）
- 用户反馈创建 Git Bash 终端时仍报 `Cannot read properties of undefined (reading 'dimensions')`
- 根因分析：
  1. 之前只 patch 了 `syncScrollArea` 和 `_refresh`，但 `_innerRefresh`（rAF回调）未 patch
  2. xterm 内部 Viewport 构造函数中 setTimeout 调用 `this.syncScrollArea()`，虽然走了 patch 入口，但 origSync 内部链路仍可能抛异常
  3. RenderService.dimensions getter 是所有链路的最终访问点，直接从根源 patch 更可靠
- 修复方案（双重保护）：
  1. **方案1（根源）**：patch `core._renderService.dimensions` getter，undefined 时返回 null 而非抛 TypeError
  2. **方案2（方法级）**：patch `syncScrollArea` + `_refresh` + `_innerRefresh` 三个方法，均加 try-catch + mountId 守卫
- **问题**：Ctrl+R / F5 刷新页面仍然无法使用
- **根因**：渲染进程 `useGlobalShortcuts.ts` 拦截了 Ctrl+R（原为"重连"操作）和 F5（原为"刷新SFTP"），都调用了 `e.preventDefault()`，导致 Electron 主进程菜单注册的 reload 快捷键被吞掉
- **修复**：开发模式下 Ctrl+R 和 F5 改为 `window.location.reload()` 刷新页面；生产模式下保留原有行为（重连/刷新SFTP）
