# 2026-05-16

## 项目分析
- 对 QSerial 项目进行了完整架构分析
- 项目为 Electron 28 + React 18 + TypeScript 5 的 pnpm monorepo
- 三个子包：@qserial/shared（类型契约）、@qserial/main（主进程）、@qserial/renderer（渲染进程）
- 核心特性：多协议终端（PTY/Serial/SSH/Telnet）、内置 MCP AI 服务器（13个工具）、文件传输服务（SFTP/FTP/TFTP/NFS）、连接共享
- 状态管理：Zustand + Immer，Tab > Session 层级模型
- IPC 通信：contextBridge 暴露类型化 API，~50 个 IPC channel
- 构建：Vite 5 + electron-builder，支持 Win/Mac/Linux 打包

## 修复 release 包主题不生效问题
- 问题：用户运行 `release/win-unpacked/QSerial.exe` 时仍显示旧紫色主题
- 根因：asar 中的代码已是最新（包含 GitHub Dark 主题和正确 CSS 变量），但 Zustand persist 中间件从 localStorage 恢复了旧主题 ID（如 `default-dark`），覆盖了 CSS 变量
- 修复：将 persist storage key 从 `"qserial-theme"` 改为 `"qserial-theme-v2"`，使旧缓存失效
- 同时在源码 `theme.ts` 中增加了 `onRehydrateStorage` 回退逻辑（找不到主题时回退到 GITHUB_DARK_THEME）
- 直接修改了 asar 中的 index.js 并重新打包（因为 Windows 环境缺少 esbuild win32 二进制无法运行 vite build）
- 注意：node_modules 是在 Linux 环境安装的，esbuild 只有 linux-x64 二进制，需要 `pnpm install` 重新安装才能在 Windows 上构建
