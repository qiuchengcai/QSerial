# QSerial 项目文件说明

## 项目概述

QSerial 是一个基于 Electron + React + TypeScript 开发的串口调试工具，支持串口、SSH、Telnet 等多种连接方式。

## 关键文件说明

### 1. 离线开发相关

| 文件 | 用途 | 使用场景 |
|------|------|----------|
| `install-deps.bat` | 下载所有依赖到 `.pnpm-store` | **外网机器**运行一次 |
| `build-offline.bat` | 离线编译项目，生成安装包 | **内网机器**运行 |
| `docs/OFFLINE_SETUP.md` | 离线环境配置详细文档 | 参考文档 |

**离线开发流程：**
1. 外网机器运行 `install-deps.bat` 下载依赖
2. 将整个 `QSerial` 文件夹复制到内网
3. 内网机器运行 `build-offline.bat` 编译

| 文件 | 用途 |
|------|------|
| `dev.bat` | 启动开发模式（热更新） |

| 文件 | 用途 |
|------|------|

## packages 目录详解

### main（主进程）

负责 Electron 主进程逻辑：
- **config/**: 配置管理（读写配置文件）
- **connection/**: 连接实现
  - `serial.ts` - 串口连接
  - `ssh.ts` - SSH 连接
  - `telnet.ts` - Telnet 连接
  - `pty.ts` - 伪终端
  - `serialServer.ts` - 串口服务器
  - `factory.ts` - 连接工厂
- **ipc/**: IPC 通信处理（主进程与渲染进程通信）
- **tftp/**: TFTP 服务器实现

### renderer（渲染进程）

负责 UI 界面：
- **components/**: React 组件
  - `terminal/TerminalPane.tsx` - 终端面板（核心组件）
  - `dialogs/*.tsx` - 各种连接对话框
  - `layout/*.tsx` - 布局组件（侧边栏、状态栏等）
- **stores/**: Zustand 状态管理
  - `sessions.ts` - 会话状态
  - `terminal.ts` - 终端状态
  - `config.ts` - 配置状态

### shared（共享代码）

主进程和渲染进程共享：
- **constants/**: 常量定义（IPC 通道、连接类型等）
- **types/**: TypeScript 类型定义
- **utils/**: 工具函数（编码、UUID、验证等）

## 使用说明

### 外网机器（准备离线包）

```powershell
cd D:\QPrj\QSerial
.\install-deps.bat
```

运行后会：
1. 检查 Node.js 和 pnpm
2. 配置 pnpm 存储到项目目录
3. 使用国内镜像下载依赖
4. 安装所有 node_modules

### 内网机器（编译项目）

```powershell
cd D:\QPrj\QSerial
.\build-offline.bat
```

运行后会：
1. 清理旧构建
2. 配置 pnpm 使用本地存储
3. 离线安装依赖
4. 编译 TypeScript
5. 打包成 exe

交付文件：`release\QSerial-0.1.0-win-x64.exe`

---

## 注意事项

1. **`.pnpm-store/`** 和 **`node_modules/`** 是离线开发必需的，不要删除
2. **`release/`** 可以删除，每次编译会重新生成
3. 复制到内网时，确保包含所有 `node_modules` 目录
4. 内网机器需要安装 Node.js 和 pnpm（全局）
