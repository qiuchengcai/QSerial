# Uniview IPC Device Connection

## Overview

Connect to Uniview IPC network cameras through QSerial's serial port sharing service (TELNET-based). Automate device login across two shell layers: Layer 1 restricted CLI and Layer 2 full Linux root shell.

## Connection Architecture

```
User → TELNET → QSerial (10.188.42.148:8888) → Serial Port → Device Shell
```

- **QSerial Authentication**: Password-based, password is `Admin123.`
- **TELNET Negotiation**: IAC WILL ECHO, WILL SUPPRESS-GA, DO NAWS, DO TTYPE, DONT LINEMODE
- **Device**: Uniview IPC6824ER-X42-W-VH2, serial `210235CB3N0471956830`

## Login Credentials

| Layer | Prompt | Login Method |
|-------|--------|-------------|
| Layer 1 | `User@/root>` | `root` / `Admin123.` |
| Layer 2 | `root@root:~$` | Send `uniview` + Ctrl+B (`\x02\x01`) + Enter, then password generated by `scripts/a` tool |

**Critical**: Layer 2 password has only 3 attempts. On failure, must `exit` back to Layer 1 and re-enter.

## Usage

### Layer 1 (Single Command)
```bash
python3 scripts/shell.py "manuinfotool"
```

### Layer 1 (Interactive)
```bash
python3 scripts/shell.py
```

### Layer 2 (Single Command)
```bash
python3 scripts/root.py "uname -a"
```

### Layer 2 (With explicit serial number)
```bash
python3 scripts/root.py --serial 210235CB3N0471956830 "cat /proc/version"
```

### Layer 2 (Interactive)
```bash
python3 scripts/root.py
```

### Password Generation Only
```bash
./scripts/a <serial_number>
```

## Script Reference

| File | Purpose |
|------|---------|
| `scripts/connect.py` | Base connection module: TCP connect, TELNET negotiation, QSerial auth, Layer 1 login |
| `scripts/shell.py` | Layer 1 CLI access (interactive + single-command) |
| `scripts/root.py` | Layer 2 root shell access: auto-fetches serial, generates password, enters Layer 2 |
| `scripts/a` | Bash wrapper for password generation, calls `y` tool with multiple configurations |
| `scripts/y` | Binary tool that generates authorization codes; `a` script wraps this tool |

## Common Layer 1 Commands

- `manuinfotool` — Device info (model, serial number, firmware)
- `netparam` — Network parameters
- `showsysinfo` — System information

## Common Layer 2 Commands

- `uname -a` — Kernel info
- `cat /proc/version` — Kernel version
- `ps` — Process list
- `logread` — System logs
- `mount` — Mounted filesystems

## Important Notes

- The `scripts/a` script depends on `scripts/y` binary tool. Both must be present in the same directory for password generation to work.
- Layer 2 password is generated using "打开调试因子+默认时间" mode based on the device serial number.
- `_hide box` enters the **equipment test layer** (`<IPC-...-equipment>`), NOT Layer 2. Do not confuse the two.
- Commands `en`, `shell`, `sh` are NOT supported on Layer 1.
